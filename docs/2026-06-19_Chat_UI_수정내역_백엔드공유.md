# 2026-06-19 Chat UI 수정내역 백엔드 공유

## 목적

Chat 화면의 최근 수정 사항을 백엔드에서 확인할 수 있도록 정리한 문서입니다.
이번 작업은 기능 추가나 mock 로직 이식이 아니라, 기존 채팅 플로우와 LLM/API 연동 구조를 유지한 상태에서 화면 디자인과 일부 렌더링 조건을 정리한 작업입니다.

## 수정 대상

- `frontend/src/app/pages/Chat.tsx`
- 참고 타입/API 파일
  - `frontend/src/app/types/chat.ts`
  - `frontend/src/app/api/chat.ts`

## 전체 변경 방향

- `/chat` 화면 전환 이후의 디자인을 Figma Make 시안 톤에 맞춰 조정했습니다.
- 기존 UI/UX 흐름은 유지했습니다.
- 기존 LLM/API 응답 구조를 그대로 사용합니다.
- 다운로드받은 mock `Chat.tsx`의 로직은 이식하지 않았습니다.
- Manual Guide, AR Guide, 완료 처리 등 기존 기능 흐름은 유지하되 화면에 표시되는 시점만 정리했습니다.

## 주요 UI 변경 사항

### 1. Chat 화면 디자인 정리

- LG Chat 헤더, 상태 문구, 입력창, 메시지 버블, 선택 버튼, 가이드 버튼의 시각 스타일을 조정했습니다.
- Bot 메시지는 글래스 계열 카드/버블 스타일로 변경했습니다.
- User 메시지는 보라색 계열 그라데이션 버블로 변경했습니다.
- 전송 버튼은 원형 그라데이션 버튼으로 변경했습니다.
- 작은 LG 아바타 이미지를 Bot 영역에 표시하도록 유지했습니다.

### 2. 첫 진입 모션과 재진입 모션 분리

- Chat 첫 진입 또는 초기화 상태에서는 기존 Chat 전용 큰 진입 모션을 유지했습니다.
- 채팅이 이미 진행된 상태에서 `/chat`에 다시 들어오면 내부 Chat 전용 blur/y/opacity 모션은 끄고, 페이지 기본 전환처럼 보이도록 루트 전환만 적용했습니다.
- 채팅이 초기화되기 전까지 재진입 시 첫 진입 모션이 다시 재생되지 않도록 처리했습니다.

### 3. 첫 진입 화면 겹침 현상 수정

- 첫 진입 LG Chat 인트로 화면 뒤에 실제 채팅 레이어가 겹쳐 보이는 문제가 있었습니다.
- 초기 화면 또는 제품 선택 단계에서는 채팅 메시지 레이어를 렌더링하지 않도록 조건을 추가했습니다.
- 실제 채팅이 시작되었거나 기존 대화가 있는 재진입 상태에서만 채팅 레이어가 표시됩니다.

## Manual Guide / AR Guide 동작 정리

### 1. Manual Guide 자동 노출 제거

- 기존에는 Bot 메시지에 `guideOptions`가 있으면 Manual Guide 내용이 버튼 클릭 전에도 자동으로 화면에 표시되는 문제가 있었습니다.
- 이제 Manual Guide 버튼을 클릭하기 전에는 가이드 상세 내용이 노출되지 않습니다.

### 2. Manual Guide 클릭 동작

- Manual Guide 클릭 시 `/self-care` 페이지로 이동하지 않습니다.
- 현재 `/chat` 화면 안에서 LLM/API 응답으로 받은 Manual Guide 내용을 인라인으로 펼쳐 보여줍니다.
- 별도의 새 API 호출은 추가하지 않았습니다.
- 기존 Bot 메시지에 포함된 `guideOptions` 데이터를 사용합니다.

### 3. AR Guide 클릭 동작

- AR Guide는 기존처럼 AR 가이드로 이동하는 흐름을 유지합니다.
- AR 단계 정보는 `message.guideOptions`에서 파생해 사용합니다.

### 4. 완료 확인 카드 표시 조건

- 완료 확인 카드는 Manual Guide 내용이 실제로 펼쳐진 뒤에 표시되도록 정리했습니다.
- 버튼 클릭 전에는 가이드 상세와 완료 확인 카드가 먼저 보이지 않도록 했습니다.

## 백엔드 영향 사항

이번 수정으로 새 API 계약은 추가되지 않았습니다.

백엔드에서는 기존처럼 Chat 응답에 아래 구조를 유지해주면 됩니다.

- `guide_options`
- `guide_options.manual_guides`
- `guide_options.youtube_recommendations`
- `guide_options.procedure_type`
- AR 관련 guide option 데이터
- `risk_level`
- `card_policy`
- `needs_clarification`
- `service_flow_type`

프론트는 Manual Guide 버튼 클릭 시 Bot 메시지 안에 이미 들어있는 `guideOptions`를 펼쳐 보여줍니다.
따라서 Manual Guide 클릭만으로 백엔드에 새 요청을 보내는 구조는 아닙니다.

## 프론트 상태 관리 참고

- 채팅 메시지는 기존 로컬 스토리지 키를 유지합니다.
  - `CHAT_STORAGE_KEY = "chat_messages_v20260618_transition_v2"`
- Manual Guide 확장 여부는 프론트 상태로 관리합니다.
  - `expandedManualGuideIds`
- Manual Guide 상세는 기존 `guideOptions`, `guideVideo`, `extractGuideSteps` 기반으로 렌더링합니다.

## 백엔드 확인 포인트

- `guide_options` 응답 구조가 기존과 동일하게 유지되는지 확인하면 됩니다.
- Manual Guide 클릭 시 별도 API 호출이 필요하지 않으므로, 클릭 이벤트용 신규 엔드포인트는 현재 필요하지 않습니다.
- AR Guide 흐름은 기존 AR 데이터 구조를 계속 사용합니다.
- 채팅 저장 API를 붙이는 경우에도 현재 프론트의 화면 흐름과 메시지 구조를 깨지 않도록 기존 메시지 필드와 `guideOptions`를 유지하는 것이 좋습니다.
